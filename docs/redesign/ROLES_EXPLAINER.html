How roles work

Four roles

manager runs the shop. mot_tester does MOT tests. mechanic fixes cars. customer is not a real user — just phone + SMS code on the public status page.

A staff person can hold more than one role (e.g. mechanic + tester).

The flow

1. Manager sets your role in Settings → Staff
↓
2. Role is saved in a locked database table (users can't touch it)
↓
3. You log in → Supabase stamps your role(s) into your login token
↓
4. Every page reads the token → allow or redirect to /403
↓
5. Every database query also re-checks the token → backup guard

Two checks (page + database) so a bug in one can't leak through the other.
